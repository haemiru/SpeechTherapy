## 🟠 정소리 — 혀운동의 임상적 필요성

### "혀운동은 구강운동치료에서 **가장 중요한 영역** 중 하나입니다."

> 입 벌리기 게임(턱 운동)은 좋은 출발이었어요. 하지만 **조음(발음)의 핵심은 혀**입니다.
> 혀의 운동 범위(ROM)와 협응력이 부족하면, ㄹ/ㅅ/ㄴ/ㄷ 같은 음소를 정확히 만들 수 없어요.
> 많은 발달장애 아동이 **혀 근력 약화**, **혀 돌출(설소대 단축)**, **좌우 비대칭** 문제를 갖고 있습니다.

---

### 1. 치료 목표

| 목표 | 설명 | 대상 아동 |
|------|------|-----------|
| **혀 신장(Protrusion)** | 혀를 입 밖으로 내밀기 | 혀 근력 약화, 삼킴 장애 |
| **혀 거상(Elevation)** | 혀끝을 입천장(경구개)으로 올리기 | ㄴ/ㄷ/ㄹ/ㅌ 조음 전 단계 |
| **혀 좌우 이동(Lateralization)** | 혀를 좌우 입꼬리 방향으로 움직이기 | 음식 씹기(저작) 보조, 구강 인식 향상 |
| **혀 운동 협응** | 위 3가지 동작을 순서대로 수행 | 복합 구강운동 능력 향상 |

---

### 2. 난이도 체계 (5단계)

> ⚠️ 반드시 단계적으로 진행해야 합니다. 아이마다 시작 레벨이 다릅니다.

| Level | 동작 | 성공 기준 | 임상 근거 |
|-------|------|-----------|-----------|
| **L1** | 혀 내밀기만 (out) | tongueOut ≥ 0.4, 0.3초 유지 | 기본 혀 운동 범위 확인 |
| **L2** | 혀 내밀기 (out) + 되돌리기 (in) | 교대 동작, 각 0.5초 유지 | 혀 수축-이완 협응 |
| **L3** | 혀 올리기 (up) | tongueUp ≥ 0.5, 0.5초 유지 | ㄴ/ㄷ/ㄹ 조음 준비 핵심 동작 |
| **L4** | 혀 좌우 이동 (left/right) | 각 방향 0.5초 유지 | 저작 보조 + 구강 공간 인식 |
| **L5** | 복합: out → up → left → right | 4동작 연속 수행 | 구강운동 프로그래밍 능력 |

---

### 3. 세션 구성

| 항목 | 값 | 근거 |
|------|-----|------|
| **라운드 수** | 5라운드 | 입 운동과 동일, 집중력 유지 |
| **라운드 시간** | 10초 | 충분한 시도 시간 확보 |
| **라운드 간 휴식** | 3초 | 혀 근육 피로 고려 (턱보다 1초 추가) |
| **세션 시간** | 약 3~5분 | 집중력 한계 존중 |

---

### 4. 측정 지표 (IEP 추적용)

| 지표 | 단위 | 설명 |
|------|------|------|
| 혀 신장 도달률 | % | 목표 threshold 대비 최대 도달값 |
| 혀 거상 도달률 | % | tongueUp 목표 대비 도달값 |
| 혀 좌우 이동 대칭성 | 비율 | left/right 균형 (1.0이 완벽 대칭) |
| 라운드 성공 횟수 | 회/세션 | 5라운드 중 성공 횟수 |
| 최대 유지 시간 | 초 | 가장 오래 유지한 시간 |
| 세션 완료율 | % | 중도 이탈 없이 완료 |

---

### 5. 임상 주의사항

> ⚠️ 혀운동 게임에서 반드시 지켜야 할 사항입니다.

1. **혀 근육은 턱보다 피로가 빠릅니다** — 라운드 간 휴식을 반드시 3초 이상
2. **침 흘림(drooling) 가능성** — 부모 안내에 "수건 준비" 포함
3. **혀 내밀기가 과도하면 역효과** — 최대 threshold를 0.85 이하로 제한
4. **구역 반사(gag reflex) 주의** — 혀 올리기(up)에서 무리하지 않도록 안내
5. **거울 효과 활용** — 카메라 화면이 곧 거울! 아이가 자기 혀를 보면서 연습 (기존에 이미 적용)

## 🟣 이하늘 — UI/UX 디자인 의견

> 입 운동 게임과 같은 카메라 기반이니까, UI 프레임을 거의 그대로 재사용할 수 있어요!

### 디자인 방향

1. **아이콘**: 👅 (혀 이모지) — 직관적
2. **색상 톤**: 분홍+보라 계열 (입 운동의 하늘+초록과 차별화)
3. **안내 애니메이션**: 혀가 어디로 가야 하는지 화살표+애니메이션으로 표시
   - ⬆️ "혀를 위로!" → 위쪽 화살표 펄스
   - ⬇️ "혀를 아래로!" → 아래쪽 화살표
   - ⬅️➡️ "혀를 옆으로!" → 좌우 화살표
   - 😛 "혀를 내밀어!" → 앞쪽 강조
4. **게이지 표시**: 기존 원형 게이지 재사용, 색상만 변경
5. **성공 시 보상**: 혀 관련 재미있는 이펙트
   - "멋진 혀 운동!" 칭찬 텍스트
   - 뭉치가 혀를 내밀며 기뻐하는 반응

## 🔵 박건축 — 기술 아키텍처 의견

> 좋은 소식입니다. 기존 게임 파이프라인을 거의 그대로 쓸 수 있습니다.

### 기술 가능성 분석

```
현재 MediaPipe Face Landmarker 설정:
  ✅ outputFaceBlendshapes: true (이미 활성화)
  ✅ tongueOut, tongueUp, tongueLeft, tongueRight 블렌드셰이프 출력 가능
  ✅ 추가 모델이나 라이브러리 불필요
```

### 아키텍처 결정 (2가지 방안)

**방안 A: 입 운동 게임에 혀운동 라운드 추가**
- 장점: 파일 최소화, 같은 카메라 세션 공유
- 단점: 게임이 비대해지고, 입운동과 혀운동의 치료 목표가 다름

**방안 B: 독립 게임으로 분리** ✅ (채택)
- 장점: 치료 목표별 독립 추적 가능, 난이도 독립 조절
- 단점: 파일 수 증가 (하지만 기존 패턴 복사로 최소화)

> 🟠 정소리: "반드시 B로 해주세요. 입 운동(턱+입술)과 혀 운동은 치료 목표가 다릅니다.
> IEP에서도 별도 항목으로 추적해야 합니다. 하나의 게임에 섞으면 데이터 분석이 어려워져요."

## 🟢 김코드 — 구현 의견

> 기존 mouth-opening 게임의 구조를 그대로 가져오면 됩니다.
> 핵심 차이점은 센서 입력이 jawOpen → tongueOut/tongueUp으로 바뀌는 것뿐이에요.

### 구현 포인트

1. **MouthMetrics 확장** — tongueOut, tongueUp, tongueLeft, tongueRight 4개 필드 추가
2. **faceGeometry 확장** — extractTongueMetrics() 함수 추가 (기존 extractMouthMetrics 패턴 동일)
3. **useMouthMetrics 훅** — 혀 메트릭도 같이 추출하도록 수정 (같은 detection loop에서)
4. **RoundType 확장** — 'tongue-out' | 'tongue-up' | 'tongue-left' | 'tongue-right' 추가
5. **게임 페이지** — mouth-opening/play 복사 후 threshold와 sensorValue만 교체
6. **라운드 타입 로직** — Level에 따라 다른 RoundType 배열 생성

```typescript
// 예시: Level별 라운드 구성
const TONGUE_ROUND_CONFIGS = {
  1: ['tongue-out', 'tongue-out', 'tongue-out', 'tongue-out', 'tongue-out'],
  2: ['tongue-out', 'tongue-in', 'tongue-out', 'tongue-in', 'tongue-out'],
  3: ['tongue-up', 'tongue-up', 'tongue-up', 'tongue-up', 'tongue-up'],
  4: ['tongue-left', 'tongue-right', 'tongue-left', 'tongue-right', 'tongue-left'],
  5: ['tongue-out', 'tongue-up', 'tongue-left', 'tongue-right', 'tongue-out'],
};
```

> 🟠 정소리: "Level 1~2는 무조건 혀 내밀기부터 시작해야 해요.
> 혀 올리기(up)는 Level 3에서, 좌우 이동은 Level 4에서.
> 한 번에 여러 동작을 시키면 안 됩니다."
>
> 김코드: "네, Level 순서대로 잠금 해제하는 방식으로 구현할게요."

## ✅ 합의 사항

### 1. 혀운동 게임 확정 사항

| 항목 | 결정 |
|------|------|
| 게임 이름 | **혀 운동** (영문: tongue-exercises) |
| 아이콘 | 👅 |
| 독립 게임 여부 | ✅ 독립 게임 (방안 B) |
| 라운드 수 | 5라운드 |
| 라운드 시간 | 10초 |
| 라운드 간 휴식 | 3초 (혀 근육 피로 고려) |
| MVP 난이도 | Level 1 (혀 내밀기만) → 이후 업데이트로 확장 |
| 잠금 해제 조건 | 입 운동 3회 완료 후 (기존 패턴) |

### 2. 개발 우선순위 업데이트

```
MVP 1차: ✅ 게임 1 (입 운동) + 홈 + 결과 + 기록 — 완료
MVP 2차: 🔜 게임 4 (혀 운동) — NEW! ← 지금 추가
MVP 3차: 게임 2 (소리 열기구)
MVP 4차: 게임 3 (따라 말하기) + 치료사/부모 대시보드
```

> 🟠 정소리: "혀 운동이 소리 열기구보다 우선순위가 높아야 합니다.
> 구강운동(입+혀)이 탄탄해야 발성(소리)과 조음(따라 말하기)으로 넘어갈 수 있어요.
> 치료 순서는: 구강운동 → 발성 → 조음입니다."
>
> 🔵 박건축: "동의합니다. 같은 MediaPipe 카메라 파이프라인을 쓰니까 개발 비용도 적습니다."
